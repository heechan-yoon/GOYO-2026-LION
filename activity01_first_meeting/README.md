## 3. 🧱 프로젝트 수행 내용

### 3.1 부품 간 연동성을 고려한 보드 선정

웨어러블 기기와 보호자용 마스터 기기가 데이터를 주고받는 과정에서 발생할 수 있는 통신 오류를 줄이기 위해, 두 기기 모두 **ESP32-S3 계열 보드**로 통일하여 구매하기로 하였다.

이를 통해 동일한 통신 프로토콜을 사용할 수 있고, 개발 환경도 단일화할 수 있어 전체 시스템의 연동 효율을 높일 수 있다고 판단하였다.

* 웨어러블 보드와 마스터 보드 모두 ESP32-S3 계열로 통일
* 동일한 개발 환경 사용 가능
* BLE / Wi-Fi 기반 통신 구현에 유리
* 보드 간 통신 오류 가능성 감소
* 시스템 통합 및 디버깅 효율 향상

---

### 3.2 데이터 품질 확보를 위한 디지털 센서 채택

AI 학습에 사용할 소음 및 조도 데이터를 안정적으로 수집하기 위해, 노이즈에 강한 **디지털 통신 방식의 센서**를 선정하였다.

아날로그 센서보다 디지털 센서가 데이터 신뢰성 측면에서 유리하다고 판단하였고, 특히 소리 데이터와 빛 변화를 정밀하게 측정할 수 있는 센서들을 우선적으로 고려하였다.

| 센서 | 통신 방식 | 측정 대상 | 선정 이유 |
| :--- | :--- | :--- | :--- |
| INMP441 | I2S | 소음 / 고주파 음향 데이터 | 디지털 마이크로 노이즈에 강하고 AI 학습용 음향 데이터 수집에 적합 |
| VEML7700 | I2C | 조도 / 빛 변화 | 조도 변화 및 플리커 감지에 활용 가능 |
| PPG Sensor | 센서별 상이 | 심박 변화 | 외부 자극에 따른 생체 반응 확인 가능 |

---

### 3.3 사용자 환경을 고려한 배터리 이원화 전략

웨어러블 기기의 사용 시간과 착용감을 동시에 고려하기 위해, 두 종류의 배터리를 구매하여 비교 테스트하기로 하였다.

가벼운 배터리는 착용감 측면에서 유리하지만 사용 시간이 짧을 수 있고, 용량이 큰 배터리는 안정적인 전력 공급이 가능하지만 크기와 무게가 증가할 수 있다.

따라서 실제 제작 단계에서 최적의 배터리 용량을 결정하기 위해 **300mAh 모델**과 **1000mAh 모델**을 함께 테스트하기로 하였다.

| 배터리 용량 | 장점 | 단점 | 테스트 목적 |
| :--- | :--- | :--- | :--- |
| 300mAh | 가볍고 크기가 작음 | 사용 시간이 짧을 수 있음 | 착용감 중심 테스트 |
| 1000mAh | 안정적인 전력 공급 가능 | 크기와 무게 증가 | 장시간 동작 테스트 |

---

### 3.4 개발 범위의 단계적 설정

이번 단계에서는 시스템의 동작 여부를 결정하는 **보드와 센서 구성**에 집중하기로 하였다.

실제 기기를 몸에 고정하는 스트랩 선정이나 구체적인 착용감 개선에 대한 논의는 하드웨어 연동이 확인된 이후 단계로 이월하였다.

이를 통해 초기 개발 단계에서 핵심 기능 구현에 집중하고, 이후 착용성 개선을 진행하는 방식으로 개발 효율을 높이고자 하였다.

* 1차 단계: 보드 및 센서 구성 확정
* 2차 단계: 단일 센서 동작 테스트
* 3차 단계: 보드 간 무선 통신 테스트
* 4차 단계: 착용감 테스트 및 스트랩 선정
* 5차 단계: 앱 및 AI 모델 연동

## 7. 💬 프로젝트 수행 성찰

### 7.1 양성현

이번 회의는 기획서상의 개념적인 아이디어를 실제 개발 가능한 스펙으로 치환하고 기술적 기반을 확립한 자리였다.

특히 자폐인의 감각적 특성을 최우선으로 고려하여 가속도 센서를 과감히 배제한 것은 기술 자체보다 사용자를 우선한 객관적이고 타당한 결정이었다.

전체적인 시제품 제작 로드맵이 명확히 수립된 만큼, 앞으로는 감정적인 기대를 배제하고 각 팀원의 파트가 자연스럽게 통합될 수 있도록 일정 관리와 시스템 아키텍처 조율에 집중할 예정이다.

---

### 7.2 송기범

초기 부품 선정 과정을 거치며 하드웨어 리소스와 데이터 분석 알고리즘 간의 기술적 정합성을 선제적으로 확보하는 성과를 얻었다.

통신 효율과 AI 포팅의 편의성을 위해 마스터 보드를 통신 통합형으로 변경하고, 가속도 센서를 배제하여 코어 센서에 집중한 것은 시스템 통합 시 발생할 수 있는 병목 현상을 사전에 차단한 의사결정이었다.

앞으로는 확립된 하드웨어 기반 위에서 양질의 교차 검증 데이터를 수집하여 시스템의 신뢰도를 실증 수준으로 끌어올리는 데 주력할 예정이다.

---

### 7.3 윤희찬

가속도 센서 배제와 투트랙 보드 분리 아키텍처가 확정됨에 따라, 마스터 보드에서 감당해야 할 통신 규격과 데이터 수신 범위가 명확해졌다.

사용자 편의를 최우선으로 하여 하드웨어 폼팩터를 제한한 만큼, 남은 핵심 데이터들이 메인 보드로 누락 없이 전달되도록 통신 안정성을 확보하는 것이 중요하다.

앞으로는 마스터 보드의 핀맵 설계와 데이터 수신 펌웨어 최적화 역할에 집중할 예정이다.

---

### 7.4 정선호

이번 회의를 통해 자이로 센서 도입을 후순위로 미루기로 한 결정은 보드의 제한된 리소스를 중요한 데이터 수집에 집중할 수 있게 해준 판단이었다.

여러 센서를 한 번에 묶을 때 우려되었던 I2C 버스 충돌 및 전력 과부하 위험을 줄이고, 데이터 샘플링 요구량이 많은 I2S 디지털 마이크의 DMA 처리에 메모리를 안정적으로 할당할 수 있게 되어 펌웨어 설계 방향성이 명확해졌다.

비록 가속도 센서의 부재로 인해 PPG 심박 데이터의 하드웨어적 움직임 노이즈 보정 수단은 사라졌지만, 기구적인 신체 밀착 설계와 알고리즘 단의 디지털 필터링 협업을 통해 이 한계를 극복할 수 있을 것으로 판단하였다.

앞으로는 확정된 센서 결선 환경을 바탕으로 노이즈 간섭을 최소화하고, 수집된 환경 데이터를 마스터 보드로 지연 없이 전송하는 저전력 BLE 통신 최적화에 집중할 예정이다.

---

### 7.5 김영민

이번 회의를 통해 아날로그 마이크 대신 I2S 통신 기반의 INMP441 디지털 마이크를 선정함으로써, 딥러닝 학습용 데이터의 신뢰성과 노이즈 저항성을 확보할 수 있었다.

또한 마스터 보드의 16MB PSRAM 사양이 확정됨에 따라, 메모리 오버플로우에 대한 우려 없이 PC에서 학습된 CNN 모델의 TFLite 경량화 및 실시간 FFT 전처리 로직 설계에 집중할 수 있는 환경이 마련되었다.
