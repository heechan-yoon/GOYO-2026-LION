### 3.3 브레드보드 기반 음향 데이터 수집 환경 구축

INMP441은 I2S 통신을 사용하는 디지털 마이크이므로, 아날로그 마이크보다 외부 노이즈의 영향을 줄이면서 일정한 형식의 오디오 데이터를 수집할 수 있다는 장점이 있다.

이번 활동에서는 ESP32-S3와 INMP441을 I2S 방식으로 결선하고, 마이크를 통해 유입되는 소리가 MCU에 정상적으로 전달되는지 확인하였다.

```text
INMP441 Digital Microphone
 ├─ Sound Input
 ├─ I2S Data Transmission
 ↓
ESP32-S3 Wearable Board
 ├─ Raw Audio Data Reception
 ├─ Amplitude Monitoring
 ├─ Frequency Analysis
 └─ Serial Plotter Visualization
```

### 3.5 감각 과부하 유발 주파수 문헌 조사

음향 데이터를 수집하는 것만으로는 어떤 대역을 위험 신호로 판단해야 하는지 결정하기 어렵다.

따라서 자폐 스펙트럼 장애를 가진 이용자가 특정 음역대에 어떻게 반응하는지 다룬 국내외 연구 자료를 검토하고, 향후 데이터 라벨링 기준을 설정하기 위한 기초 조사를 진행하였다.

회의에서는 특정 고주파 대역을 중심으로 통제 환경 테스트와 AI 학습 데이터를 구성하는 방향을 검토하였다.

* 감각 과부하 유발 가능성이 있는 음향 대역 조사
* 고주파 관련 위험 트리거 후보 탐색
* 통제 환경 실험에서 사용할 주파수 대역 선정 방향 수립
* 정상 상태와 위험 전조 상태를 구분하기 위한 라벨링 기준 검토

> **Note:** 구체적인 위험 주파수 대역은 추후 참고 논문과 출처를 명시하고, 실제 테스트 결과와 함께 검증할 예정이다.

### 4.3 주파수 분석 알고리즘의 타당성 확인

마이크에서 수집한 Raw audio data를 기반으로 특정 주파수 대역을 실시간으로 분석할 수 있음을 확인하였다.

이를 통해 향후 웨어러블 보드가 원본 오디오 전체를 전송하는 대신, 감각 과부하와 관련된 특정 주파수 대역의 energy 값만 추출하여 마스터 보드로 전달하는 구조가 기술적으로 가능함을 검증하였다.

```text
기존 방식
Raw Audio 전체 데이터 전송
 → 데이터 용량 증가
 → 통신 부하 증가
 → 배터리 사용량 증가

개선 방향
특정 위험 주파수 대역의 Energy 값만 추출
 → 데이터 경량화
 → 통신 부하 감소
 → 웨어러블 전력 소모 감소
```

현재 확인한 주파수 분석 코드를 최적화하여, 마이크 원본 데이터 전체가 아니라 감각 과부하와 관련된 특정 주파수 대역의 energy 값만 추출하는 내부 로직을 완성할 예정이다.

* FFT 기반 주파수 분석 코드 최적화
* 특정 주파수 대역별 energy 계산
* 불필요한 Raw audio data 제거
* 노이즈 필터링 기준 설정
* 경량 데이터 패킷 구조 설계

### 6.2 추가 센서 통합 및 핀맵 검증

INMP441 마이크 외에 조도 센서와 심박 센서를 추가하여 다중 센서 수집 환경을 구성할 예정이다.

| 센서 | 통신 방식 | 측정 대상 | 다음 단계 |

| VEML7700 | I2C | 조도 / 빛 변화 | 추가 결선 및 동작 테스트 |
| MAX30102 | I2C | 심박 변화 | 추가 결선 및 동작 테스트 |

추가 센서 통합 과정에서는 I2C 핀 충돌이나 전력 부족 문제가 발생하지 않는지 확인해야 한다.

* VEML7700 단일 동작 테스트
* MAX30102 단일 동작 테스트
* INMP441과 I2C 센서 동시 동작 테스트
* 핀 충돌 여부 점검
* 다중 센서 데이터의 동시 출력 확인

## 7. 💬 프로젝트 수행 성찰

### 7.1 양성현

오늘 회의는 프로젝트가 기획서상의 텍스트를 넘어, 실제 데이터를 처리하는 물리적 하드웨어로 첫걸음을 내딛은 자리였다.

단순히 소음의 크기를 측정하는 데 그치지 않고, ESP32-S3와 INMP441 마이크를 I2S 통신으로 결선하여 특정 주파수 대역의 성분을 실시간으로 추적하는 데 성공하였다.

특히 하드웨어 동작 테스트 직후 진행한 자폐 관련 문헌 조사는 우리가 개발하는 기술의 목적성을 더욱 분명하게 만들어 주었다. 시리얼 플로터에 나타난 주파수 파형이 단순한 공학적 수치가 아니라, 향후 멜트다운 위험을 사전에 감지하기 위한 핵심 트리거로 활용될 수 있음을 확인하였다.

---

### 7.2 송기범

다른 팀원들이 하드웨어 단에서 보드를 연결하고 마이크 신호를 검증하는 동안, 수집한 데이터를 인공지능 학습용 데이터셋으로 만들기 위한 소프트웨어 환경과 데이터 수집 방안을 조사하였다.

Cursor 에디터와 Python 라이브러리를 활용하여 보드의 데이터를 실시간 CSV 파일로 저장하는 방법과 전체적인 개발 툴체인 구성을 파악하였다.

또한 문헌 조사를 바탕으로 향후 본격적인 데이터 수집이 시작될 때 어떤 데이터를 위험 전조 상태로 분류하고 라벨링해야 할지 검토하였다. 앞으로는 실제 노트북에 데이터 수집 환경을 구축하고, 부품이 정상 작동하는 대로 데이터를 수집하여 전처리 로직을 구체화할 예정이다.

---

### 7.3 윤희찬

이번 회의를 통해 프로젝트가 단순한 아이디어 단계에서 벗어나 실제 하드웨어와 센서를 기반으로 데이터를 수집하고 분석하는 단계로 나아갔다는 점이 가장 인상 깊었다. ESP32-S3 보드와 INMP441 마이크를 I2S 통신으로 연결하여 소리의 진폭뿐만 아니라 특정 주파수 대역까지 실시간으로 확인할 수 있었고, 이를 통해 자폐인의 감각 과부하를 유발할 수 있는 환경 요인을 공학적으로 감지할 수 있다는 가능성을 확인했다. 

특히 단순히 센서 값을 받아오는 것에 그치지 않고, 문헌 조사를 통해 실제로 어떤 주파수 대역을 위험 신호로 설정해야 하는지 기준을 찾아가는 과정에서 이 프로젝트가 기술 구현뿐만 아니라 사용자에게 실질적인 도움을 주는 방향으로 설계되어야 한다는 점을 느꼈다. 앞으로는 마이크 데이터 전처리 알고리즘을 더 안정화하고, 조도 및 심박 센서까지 함께 통합하여 다양한 생체·환경 정보를 기반으로 멜트다운을 사전에 예측할 수 있는 시스템을 구현하는 데 기여하고 싶다.

---

### 7.4 정선호

이번 회의에서는 브레드보드 상에 ESP32-S3와 INMP441 디지털 마이크를 I2S 통신으로 결선하고, 시리얼 플로터를 통해 오디오 파형이 지연 없이 실시간으로 출력되는 것을 직접 구현하였다.

특히 마이크의 Raw 데이터를 단순히 받아오는 것을 넘어 실시간 주파수 분석까지 성공적으로 테스트한 점이 의미 있었다.

이번 음향 기초 세팅을 기반으로 향후 조도 및 심박 센서를 핀 충돌 없이 I2C 방식으로 안정적으로 통합하고, 추출한 다중 센서 데이터 패킷을 마스터 보드로 끊김 없이 전달하기 위한 ESP-NOW 또는 BLE 기반 무선 통신망 구축에 집중할 예정이다.

---

### 7.5 김영민

이번 회의와 실습을 통해 ESP32-S3 보드와 디지털 마이크를 연결하고, 소리 데이터를 수집하여 실시간으로 주파수를 분석하는 핵심 파이프라인을 구축하였다.

단순히 소음의 크기만 측정하는 것을 넘어, 보드 내부에서 오디오 데이터를 주파수 성분으로 분해하는 과정을 직접 구현한 점이 가장 큰 성과였다.

특히 하드웨어 테스트 과정에서 발생하는 미세한 소음과 신호 흔들림을 제어하기 위해 소프트웨어 단에서 특정 수치 이하의 노이즈를 걸러내는 필터링 로직을 반영하였다. 이를 통해 데이터의 신뢰성을 높이는 전처리 과정의 중요성을 확인하였다.

실제 스마트폰 음원을 사용하여 고주파 신호를 보드가 실시간으로 탐지하는 모습을 확인하면서, 초기 기획한 시스템 구조의 실현 가능성을 1차적으로 검증하였다. 앞으로는 수집한 데이터를 기반으로 인공지능 모델 학습용 데이터셋을 구축하고 성능을 최적화할 예정이다.

또한 Raw audio data 전체를 마스터 보드로 전송하는 대신, 특정 위험 주파수 대역의 energy 값만 추출하여 전달하는 경량화 구조의 가능성을 확인하였다.
