### 3.1 XIAO ESP32-S3 웨어러블 보드 중심 구조로 이전

이전 활동에서는 센서의 기본 동작을 빠르게 검증하기 위해 ESP32-S3 DevKit 보드를 중심으로 테스트를 진행하였다.

이번 활동에서는 실제 웨어러블 디바이스 제작을 고려하여, 보다 작은 크기의 **XIAO ESP32-S3 보드**로 센서 테스트 환경을 이전하였다.

XIAO ESP32-S3는 크기가 작아 착용형 디바이스 제작에 적합하지만, 연결 가능한 핀의 수와 배선 공간이 제한적이다. 따라서 센서별 통신 방식과 핀맵을 정리하고, 다중 센서 연결 시 발생할 수 있는 충돌 가능성을 확인하는 과정이 필요하였다.

```text
Previous Test Environment
 ├─ ESP32-S3 DevKit
 ├─ Breadboard-based Connection
 └─ Individual Sensor Verification

Wearable Test Environment
 ├─ XIAO ESP32-S3
 ├─ Compact Form Factor
 ├─ Soldered Pin Headers
 ├─ INMP441 Microphone
 ├─ VEML7700 Light Sensor
 └─ MAX30102 Heart-Rate Sensor
```

### 3.4 INMP441 마이크 센서 동작 검증

Activity 02에서 ESP32-S3 DevKit과 연결하여 테스트했던 INMP441 마이크 센서를 XIAO ESP32-S3 웨어러블 보드에 직접 연결하였다.

INMP441은 I2S 통신을 사용하는 디지털 마이크이므로, 웨어러블 보드에서도 음향 데이터를 안정적으로 수집할 수 있는지 확인해야 했다.

테스트 결과, I2S 데이터가 정상적으로 출력되었으며 주변 소음과 순간적인 음향 변화를 구분할 수 있음을 확인하였다.

특히 박수 소리를 입력했을 때 signal 값이 뚜렷하게 상승하는 것을 확인하였다.

* INMP441과 XIAO ESP32-S3 간 I2S 연결
* Raw audio data 출력 확인
* Baseline 기반 음향 전처리 적용
* 주변 소음과 순간적인 음향 변화 구분
* 박수 입력 시 signal 값 상승 확인

```text
INMP441 Microphone
 ├─ Raw Audio Input
 ├─ I2S Communication
 ↓
XIAO ESP32-S3
 ├─ Baseline Estimation
 ├─ Noise Filtering
 ├─ Signal Variation Detection
 └─ Serial Monitor Output
```

### 3.5 VEML7700 조도 센서 동작 검증

VEML7700 조도 센서를 XIAO ESP32-S3의 I2C 환경에 연결하여 외부 밝기 변화를 실시간으로 측정할 수 있는지 확인하였다.

테스트 결과, 외부 빛 변화에 따라 `lux`, `white`, `als` 값이 실시간으로 변화하는 것을 확인하였다.

이를 통해 주변 환경의 밝기 변화를 정량적으로 측정할 수 있는 기초 데이터 수집 환경을 구축하였다.

| 출력값 | 의미 | 활용 방향 |
| :--- | :--- | :--- |
| `lux` | 조도값 | 주변 밝기 수준 분석 |
| `white` | 백색광 관련 측정값 | 빛 변화 추적 |
| `als` | Ambient Light Sensor Raw value | 조도 데이터 변화 분석 |

* VEML7700과 XIAO ESP32-S3 간 I2C 연결
* I2C 주소 인식 여부 확인
* 외부 빛 변화에 따른 `lux`, `white`, `als` 값 변화 확인
* 조도 변화 실시간 출력 검증
* 향후 밝기 급변 및 플리커 관련 위험도 분석 기반 마련

### 3.6 MAX30102 심박 센서 동작 검증

MAX30102 심박 센서를 XIAO ESP32-S3의 I2C 환경에 연결하고, 손가락 접촉 여부에 따라 Raw data가 변화하는지 확인하였다.

테스트 결과, MAX30102가 정상적으로 인식되었으며 손가락을 센서에 접촉했을 때 `red`, `ir` Raw value가 변화하는 것을 확인하였다.

다만 안정적인 BPM 값을 산출하기 위해서는 측정 환경과 알고리즘을 추가로 개선해야 한다.

| 출력값 | 의미 | 현재 상태 |
| :--- | :--- | :--- |
| `red` | Red LED 기반 PPG Raw value | 손가락 접촉에 따른 변화 확인 |
| `ir` | IR LED 기반 PPG Raw value | 손가락 접촉에 따른 변화 확인 |
| `BPM` | Beats Per Minute | 안정화를 위한 추가 개선 필요 |

* MAX30102와 XIAO ESP32-S3 간 I2C 연결
* 센서 인식 여부 확인
* 손가락 접촉에 따른 `red`, `ir` Raw value 변화 확인
* PPG Raw data 수집 가능성 확인
* 안정적인 BPM 산출을 위한 추가 알고리즘 개선 필요

### 3.7 음향·조도·심박 센서 종합 테스트

각 센서의 개별 동작을 확인한 뒤, XIAO ESP32-S3 웨어러블 보드 환경에서 음향·조도·심박 데이터를 함께 확인하는 종합 테스트를 진행하였다.

이번 종합 테스트에서는 세 가지 센서의 출력 상태를 점검하고, 향후 하나의 통합 코드에서 센서 데이터를 동시에 수집할 수 있는지 검토하였다.

```text
XIAO ESP32-S3 Wearable Board
 ├─ INMP441 Microphone
 │   └─ Audio Signal Value
 │
 ├─ VEML7700 Light Sensor
 │   └─ Lux / White / ALS Value
 │
 └─ MAX30102 Heart-Rate Sensor
     └─ Red / IR Raw Value
```

| 센서 | 통신 방식 | 주요 출력값 | 현재 검증 상태 |
| :--- | :--- | :--- | :--- |
| INMP441 | I2S | Audio signal | 정상 출력 확인 |
| VEML7700 | I2C | `lux`, `white`, `als` | 정상 출력 확인 |
| MAX30102 | I2C | `red`, `ir` | Raw data 변화 확인 |
| MAX30102 | I2C | `BPM` | 추가 안정화 필요 |

### 3.8 감각 과부하 위험 판단 기준 구체화

하드웨어 테스트와 함께 자폐인의 감각 과부하를 유발할 수 있는 환경 조건을 조사하였다.

단일 센서값만으로 위험 상황을 판단하기보다는 음향, 조도, 플리커, 심박 변화가 함께 나타나는 상황을 분석하는 방향으로 시스템을 설계하기로 하였다.

#### 음향 데이터 분석 방향

* 특정 주파수 대역의 energy 추출
* 약 250Hz 부근 저주파 대역 분석
* 약 6,000~8,000Hz 고주파 대역 분석
* 약 65~75dB 범위의 소음 지속 시간 측정
* 일반 생활 소음과 위험 가능성이 있는 음향 조건 구분

#### 조도 및 플리커 분석 방향

* 주변 조도 변화량 측정
* 갑작스러운 밝기 변화 감지
* 조도와 플리커 조건의 복합 위험도 분석
* 단순 밝기 수치뿐만 아니라 시간에 따른 변화 패턴 분석

#### 심박 데이터 분석 방향

* 환경 자극 발생 시점 기록
* 심박 변화 발생 시점 기록
* 음향·조도 자극과 심박 변화의 시간축 정렬
* 자극 발생 전후 PPG Raw data 및 BPM 변화 분석

> **Note:** 위 조건은 향후 데이터 수집 및 알고리즘 설계를 위한 초기 기준이다. 실제 위험 판단 threshold는 참고 자료 검토와 반복 테스트를 통해 추가 검증할 예정이다.

## 6. 🚧 앞으로의 과제

### 6.1 MAX30102 BPM 및 PPG 파형 안정화

MAX30102의 `red`, `ir` Raw value 변화는 확인하였지만, 안정적인 BPM 산출을 위해 추가 개선이 필요하다.

* 손가락 고정 방식 개선
* LED 밝기 조절
* 측정 시간 조정
* Sampling condition 조정
* Moving average 기반 노이즈 제거 적용
* Peak detection 알고리즘 개선
* 평균 심박수 및 BPM 안정성 검증

---

### 6.2 VEML7700 및 MAX30102 동시 I2C 연결

VEML7700과 MAX30102는 모두 I2C 통신을 사용한다.

두 센서를 하나의 XIAO ESP32-S3 보드에 연결하고, 주소 충돌 없이 동시에 데이터를 출력할 수 있는지 확인해야 한다.

| 센서 | I2C 주소 | 확인 항목 |
| :--- | :--- | :--- |
| VEML7700 | `0x10` | 조도 센서 인식 여부 |
| MAX30102 | `0x57` | 심박 센서 인식 여부 |

* VEML7700의 `0x10` 주소 인식 여부 확인
* MAX30102의 `0x57` 주소 인식 여부 확인
* 하나의 XIAO ESP32-S3에서 두 센서 동시 동작 검증
* `lux`, `red`, `ir` 데이터를 하나의 시리얼 모니터에서 출력
* I2C 통신 오류 및 핀 충돌 여부 확인

---

### 6.3 다중 센서 통합 코드 작성

INMP441, VEML7700, MAX30102의 데이터를 하나의 코드에서 동시에 수집할 수 있도록 통합 코드를 작성할 예정이다.

```text

 ├─ Audio Signal Value
 ├─ Lux Value
 ├─ Red Raw Value
 ├─ IR Raw Value
 └─ Timestamp
```

* INMP441 마이크 센서의 signal 값 출력
* VEML7700 조도 센서의 `lux` 값 출력
* MAX30102 심박 센서의 `red`, `ir` Raw value 출력
* 센서별 sampling period 정리
* Timestamp 기반 데이터 정렬
* CSV 저장을 고려한 출력 형식 통일

### 6.5 센서 융합 기반 위험 판단 로직 구체화

단일 센서의 threshold만으로 위험 상황을 판단하지 않고, 음향·조도·심박 변화의 상관관계를 함께 분석하는 구조를 설계할 예정이다.

* 약 65~75dB 범위의 소음 지속 시간 측정

## 7. 💬 프로젝트 수행 성찰

### 7.1 양성현

자폐인의 시각적 감각 과부하와 관련된 조도 및 플리커 조건을 조사하였다.

조사 과정에서 단순히 주변 밝기만 측정하는 것이 아니라, 조도와 플리커 주파수가 동시에 위험 수준에 도달하는 상황을 고려해야 함을 확인하였다.

향후 실제 센서 측정값을 바탕으로 복합 위험 판단 기준을 구체화할 예정이다.

---

### 7.2 송기범

자폐인의 감각 과부하를 유발할 수 있는 음향 조건을 조사하였다.

단순한 소음 크기뿐만 아니라 특정 주파수 대역과 소음 지속 시간도 위험 판단에 중요한 요소임을 확인하였다.

이후 실제 환경에서 음향 데이터를 수집하고 FFT 분석을 적용하여 일반 생활 소음과 위험 가능성이 있는 소음을 구분할 수 있는 기준을 마련할 계획이다.

---

### 7.3 윤희찬

조도 센서, 심박 센서, 웨어러블 전용 보드의 납땜 작업을 진행하였다.

센서 테스트를 소프트웨어 단계에서 끝내지 않고 실제 웨어러블 형태로 구현하기 위해서는 하드웨어 조립의 완성도도 중요하다는 점을 확인할 수 있었다.

향후에는 착용 편의성과 배선 안정성을 고려하여 센서 배치와 연결 구조를 개선할 예정이다.

---

### 7.4 정선호

XIAO ESP32-S3 웨어러블 보드와 센서 모듈의 납땜 작업을 진행하였다.

작은 크기의 보드에 여러 센서를 안정적으로 연결하기 위해서는 정확한 핀 배치와 결선 구조가 중요함을 확인하였다.

다음 단계에서는 센서별 결선 상태를 점검하고, 다중 센서 연결 시 발생할 수 있는 오류를 줄이는 데 집중하고자 한다.

---

### 7.5 김영민

기존 마스터 보드 중심의 센서 테스트 환경을 XIAO ESP32-S3 웨어러블 보드 기반 구조로 이전하였다.

INMP441 마이크, VEML7700 조도 센서, MAX30102 심박 센서의 개별 데이터 수집이 가능함을 확인하였으며, 심박수 산출 안정화와 다중 센서 통합이 추가로 필요함을 파악하였다.

## 10. 📌 최종 요약

이번 3차 회의에서는 기존 ESP32-S3 DevKit 중심의 센서 테스트 환경을 XIAO ESP32-S3 웨어러블 보드 기반 구조로 이전하였다.
