# ⚡ GOYO 2026 Academic Town

> **프로그램:** 2026 한양학술타운 LION  
> **팀명:** 고요(GOYO)  
> **프로젝트명:** 자폐인과 보호자를 위한 스마트 웨어러블 디바이스 및 시스템  
> **주제:** 자폐인의 멜트다운 예방을 위한 주변 환경 및 실시간 생체 변화 기반 맞춤형 알림 디바이스 개발

---

## 1. 🎯 프로젝트 목표

**GOYO**는 감각 과부하로 고통받는 자폐인들에게 자극 없는 **고요하고 평온한 환경**을 찾아주기 위한 스마트 웨어러블 시스템 개발 프로젝트입니다.

본 프로젝트의 핵심 목표는 자폐인이 일상 속에서 겪을 수 있는 감각 과부하와 멜트다운 위험을 사전에 감지하고, 보호자가 빠르게 대응할 수 있도록 돕는 것입니다.

* **목표 1:** 주변 환경 자극과 생체 신호를 실시간으로 측정하여 감각 과부하 위험 징후를 조기에 감지
* **목표 2:** 자폐인의 멜트다운 또는 패닉 상황이 발생하기 전에 보호자에게 위험 알림 제공
* **목표 3:** 웨어러블 디바이스의 무게와 발열을 최소화하여 착용 스트레스 감소
* **목표 4:** 환경 데이터와 생체 데이터를 융합 분석하는 Sensor Fusion 기반 예방 시스템 구현
* **목표 5:** 단순 사후 대응이 아닌 데이터 기반 사전 예방형 헬스케어 디바이스 구현

---

## 2. ❓ 해결하고자 하는 질문

본 프로젝트는 다음과 같은 문제의식에서 출발했습니다.

1. **왜 보호자가 곁에 있어도 자폐인의 멜트다운을 예측하고 막지 못할까?**
2. **일반 사람이 인지하기 어려운 감각 과부하 트리거를 센서로 측정할 수는 없을까?**
3. **생체 데이터와 주변 환경 데이터를 함께 분석하면 위기 상황을 사전에 예측할 수 있을까?**

자폐인은 큰 소음, 갑작스러운 빛 변화, 특정 주파수의 고주파 소음 등으로 인해 감각 과부하를 겪을 수 있습니다. 하지만 이러한 자극은 일반인이 직관적으로 인지하기 어렵기 때문에, 보호자가 곁에 있어도 위기 상황을 사전에 알아차리기 어렵습니다.

따라서 본 프로젝트는 **보이지 않는 트리거를 데이터로 측정하고, 위기 전조를 조기에 포착하는 시스템**을 구현하는 것을 목표로 합니다.

---

## 3. 🧩 전체 시스템 구조

본 프로젝트는 웨어러블 디바이스와 마스터 보드를 분리한 **Two-track Device Architecture**를 기반으로 합니다.

```text
Wearable Device
 ├─ ESP32-S3 기반 센서 수집 보드
 ├─ VEML7700 조도 / 플리커 센서
 ├─ INMP441 소음 / 고주파 마이크
 ├─ PPG 심박 센서
 └─ BLE 기반 센서 데이터 송신

Master Board
 ├─ 웨어러블 센서 데이터 수신
 ├─ FFT 기반 음향 신호 처리
 ├─ 1D-CNN 기반 위험 상황 예측
 ├─ 환경 데이터 + 생체 데이터 융합 분석
 └─ 보호자 스마트폰 앱으로 위험 알림 전송
```

웨어러블 디바이스는 데이터 수집과 BLE 송신에 집중하고, 무거운 연산은 마스터 보드가 담당합니다. 이를 통해 사용자가 착용하는 디바이스의 크기, 무게, 배터리 부담을 줄이고자 합니다.

## 4. 🧱 핵심 구성 요소

### 4.1 초경량 웨어러블 디바이스

자폐인의 촉각 민감성을 고려하여, 웨어러블 디바이스에는 복잡한 연산 장치를 탑재하지 않고 센서 데이터 수집 및 송신 기능만 포함합니다.

* ESP32-S3 기반 센서 수집
* BLE 기반 데이터 송신
* 저전력 동작
* 배터리 용량 및 기기 부피 최소화
* 착용 시 이물감과 발열 최소화

---

### 4.2 환경 데이터 수집

감각 과부하를 유발할 수 있는 외부 환경 자극을 정량적으로 측정합니다.

| 센서 | 측정 대상 | 목적 |
| :--- | :--- | :--- |
| VEML7700 | 조도, 빛 변화, 플리커링 | 갑작스러운 빛 변화 및 플리커 감지 |
| INMP441 | 소음, 고주파, 데시벨 | 특정 주파수 대역의 소음 및 고주파 자극 감지 |
| PPG Sensor | 심박 변화 | 외부 자극에 따른 생체 반응 확인 |

---

### 4.3 생체 데이터 수집

외부 자극이 사용자에게 실제로 어떤 생체 반응을 유발하는지 확인하기 위해 심박 데이터를 함께 수집합니다.

* 심박수 변화 추이 측정
* 환경 변화와 생체 반응의 상관관계 분석
* 단순 소음 감지가 아닌 사용자 반응 기반 위험 판단

---

### 4.4 AI 기반 위험 예측

단순 임계값 기반 알림은 오작동 가능성이 높기 때문에, 다중 센서에서 수집된 시계열 데이터를 분석하는 **1D-CNN 모델**을 적용합니다.

* 환경 신호와 생체 신호의 변화 패턴 학습
* FFT를 이용한 고주파 대역 에너지 추출
* 15~20kHz 대역의 음향 특성 분석
* 일상 소음과 실제 위기 트리거 소음 구분
* 위험 상황 판단 결과를 보호자 앱으로 전송

## 6. 🔬 연구 및 개발 방법

### 6.1 하드웨어 및 센서 인터페이스 설계

* 마스터 보드와 ESP32-S3 기반 웨어러블 보드를 분리하여 구성
* VEML7700 센서는 I2C 통신 기반으로 연결
* INMP441 마이크는 I2S 통신 기반으로 연결
* PPG 센서를 이용해 심박 데이터 수집
* 보드 간 데이터 송수신을 위한 BLE 통신 구조 설계

---

### 6.2 신호 처리 및 AI 모델링

* 수집된 음향 시계열 데이터에 FFT 적용
* 고주파 대역의 에너지 변화 추출
* 플리커, 고주파, 심박수 데이터를 기반으로 CNN 모델 학습
* 학습된 모델을 경량화하여 마스터 보드에 탑재
* 환경 데이터와 생체 데이터의 상관관계를 기반으로 위험 상황 판단

---

### 6.3 무선 통신 및 앱 기반 알림

* 웨어러블 보드와 마스터 보드 간 BLE 통신 구현
* 센서 데이터 패킷 구조 설계
* 마스터 보드에서 위험 판단 후 보호자 스마트폰 앱으로 알림 전송
* MQTT 또는 HTTP 방식 기반의 알림 시스템 구현 예정

## 11. 📝 활동 기록 방식

본 레포지토리는 프로젝트 진행 과정에서 발생하는 회의, 개발, 실험, 디버깅 내용을 기록하기 위한 저장소입니다.

* 모든 활동 기록은 `records/` 폴더에 회차별로 정리합니다.
* 회의 내용과 개발 로그는 분리하지 않고 하나의 활동기록으로 작성합니다.
* 하드웨어 조립 과정, 회로 결선도, 시리얼 모니터 출력, 트러블슈팅 과정은 반드시 기록합니다.
* 실패한 실험도 삭제하지 않고 원인과 해결 과정을 남깁니다.
* 활동 사진과 캡처 자료는 `assets/images/` 폴더에 저장합니다.
